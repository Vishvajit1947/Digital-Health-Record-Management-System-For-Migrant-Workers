You are building a production-level frontend for an NFC-Based Digital Health Record 
Management System for migrant workers. The stack is React + Tailwind CSS + Supabase JS 
client. Use Vite as the build tool. All UI must be responsive, accessible, and 
dark-mode compatible.

=====================================================================
TECH STACK
=====================================================================
- React 18 + Vite
- Tailwind CSS (utility-first styling)
- React Router v6 (routing)
- Supabase JS (@supabase/supabase-js) for auth + data
- react-i18next for multilingual support
- recharts for charts and graphs
- react-leaflet for region maps
- react-hot-toast for notifications
- lucide-react for icons
- Web NFC API (navigator.nfc.scan) for NFC tap on doctor interface

=====================================================================
AUTHENTICATION & ROUTING
=====================================================================
Implement role-based routing. After login, redirect users based on their role:
  - role = "worker"  → /worker/dashboard
  - role = "doctor"  → /doctor/dashboard
  - role = "admin"   → /admin/dashboard

Protect all routes with a <RoleGuard> component that reads the JWT role 
from Supabase session. Unauthenticated users are redirected to /login.

Routes:
  /login
  /register
  /worker/dashboard
  /worker/records
  /worker/prescriptions
  /worker/notifications
  /doctor/dashboard
  /doctor/scan        ← NFC scan page
  /doctor/patient/:id
  /doctor/add-record/:patientId
  /admin/dashboard
  /admin/analytics
  /admin/risk-table

=====================================================================
SHARED COMPONENTS (used across all roles)
=====================================================================
1. <Navbar>
   - Shows logo "HealthID" on left
   - Role badge (Worker / Doctor / Admin) in center
   - Logout button + language selector on right
   - Dark mode toggle

2. <Sidebar>
   - Collapsible left sidebar
   - Links change based on role
   - Worker: Dashboard, My Records, Prescriptions, Notifications
   - Doctor: Dashboard, Scan Patient, My Patients
   - Admin: Dashboard, Analytics, Risk Table

3. <StatCard title value icon color>
   - Used for summary metrics on all dashboards
   - Shows a large number, label, and colored icon
   - Example: "Total Workers — 4,820"

4. <HealthScoreMeter score>
   - Circular arc gauge (SVG-based)
   - Score 0–100 with color: green (>70), amber (40–70), red (<40)
   - Shows score number in center

5. <RoleGuard role>
   - Wrapper component that checks Supabase session role
   - Redirects to /login if not authenticated or wrong role

6. <LoadingSpinner>
7. <ErrorBoundary>
8. <ToastNotification> (via react-hot-toast)

=====================================================================
PAGE 6: NFC SCAN PAGE (/doctor/scan)
=====================================================================
This is the most important page.

Layout: Full-screen centered card

State 1 — Ready to scan:
  - Large NFC icon (animated pulse ring)
  - Text: "Hold patient's NFC card near device"
  - "Start Scan" button that calls navigator.nfc.scan()
  - Fallback: manual Health ID input field + "Search" button

State 2 — Scanning (loading):
  - Spinner with text "Reading NFC tag..."

State 3 — Patient found:
  - Patient profile card slides in:
      Photo placeholder, Full name, Health ID, Age, Blood Type, Region
      HealthScoreMeter
      Risk level badge
  - Two action buttons:
      "View Full History" → /doctor/patient/:id
      "Add New Record" → /doctor/add-record/:patientId

State 4 — Not found:
  - Error message: "No patient found for this NFC tag"
  - Retry button

On NFC scan: read NDEF text record → extract token → 
call Supabase to lookup nfc_tokens → get worker_id → fetch worker profile

PAGE 8: ADD RECORD PAGE (/doctor/add-record/:patientId)
=====================================================================

Section 2 — Diagnosis:
  - Diagnosis text (textarea)

  - Clinical notes (textarea)

Section 3 — Prescription (repeatable):
  - "Add Prescription" button adds a new row
  - Each row: Drug Name, Dosage, Frequency, Duration (days), Remove button

Section 4 — Lab Reports:

  - Shows uploaded file names

Submit button:

  - POST to prescriptions (one per row)
  - POST file URLs to lab_reports
  - Trigger SMS notification to worker via backend /notify endpoint
  - Show success toast, redirect to patient page

=====================================================================
SUPABASE INTEGRATION NOTES
=====================================================================
- Create a single supabase.js client file:
    import { createClient } from '@supabase/supabase-js'
    export const supabase = createClient(VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY)

- Use supabase.auth.onAuthStateChange() in App.jsx to maintain session
- Store user role in React Context (AuthContext)
- All data fetching: use custom hooks (useFetch, useWorkerData, useHealthRecords)
- Enable Realtime on health_records table for live updates in doctor view

DESIGN SYSTEM

Color tokens (add to tailwind.config.js):

  warning: #D97706 (amber)
  danger:  #DC2626 (red)

Component rules:
  - Cards: bg-white rounded-2xl shadow-sm border border-slate-100 p-6
  - Buttons primary: bg-primary text-white rounded-xl px-5 py-2.5 hover:bg-indigo-700
  - Buttons outline: border border-slate-200 rounded-xl px-5 py-2.5 hover:bg-slate-50
  - Inputs: border border-slate-200 rounded-xl px-4 py-2.5 focus:ring-2 focus:ring-primary
  - Badges: rounded-full px-3 py-1 text-xs font-medium
    - Low risk:      bg-green-100 text-green-700
    - Moderate risk: bg-amber-100 text-amber-700
    - High risk:     bg-red-100 text-red-700
    - Critical:      bg-red-600 text-white

Dark mode: use Tailwind dark: prefix throughout. 
  dark:bg-slate-900, dark:text-slate-100, dark:border-slate-700

Animations:
  - NFC scan pulse: animate-ping on the ring around NFC icon
  - Page transitions: fade-in on route change (use framer-motion or CSS)
  - Card hover: hover:-translate-y-0.5 transition-transform

Spacing: consistent 6/8/12/16/24/32px gaps using Tailwind spacing scale
